## 四、首批 100 个用户的奖励机制

### 方案 A：前 100 名注册 Pro 终身五折（推荐，启动最快）

**适用场景**：上线当天 0 用户基数，需要第一波注册。

**话术模板**

> 你是前 100 位用户之一 🎉
>
> 作为感谢，你将获得 **Pro 订阅终身 5 折** 特权。
> 原价 19.9/月 -> 你只需 9.9/月。
>
> 这个价格只限前 100 人，用完即涨。
>
> 👉 [立即升级 Pro]

**操作流程**

1. 在注册成功页或欢迎邮件中展示上述话术
2. 后端逻辑：用一个计数器 `signup_count`（Redis 存），每次注册 +1
3. `signup_count <= 100` 时，该用户的 upgrade 链接附带 `discount=50` 参数
4. Lemon Squeezy 后台创建一个折扣码（或直接在 `checkout_data` 中设置折扣）
5. 当 `signup_count` 达到 100 时，页面自动切换为标准价格

**代码级实现要点**

```ts
// 在注册接口中
const signupCount = await redis.incr("signup_count");
const isEarlyAdopter = signupCount <= 100;

const { url } = await createCheckout({
  variantId: PRO_VARIANT_ID,
  userId: newUser.id,
  redirectUrl: `${origin}/dashboard`,
});
```

### 方案 B：推荐 3 个好友升级送 1 个月 Pro（适合有一定基础用户后开启）

**适用场景**：产品已有约 50 个自然注册用户后启动，利用社交裂变。

**话术模板（推荐人看到）**

> 你已成功邀请 **{n}** 位好友 🎉
> 再邀请 {3-n} 位就能免费获得 1 个月 Pro！
> 👉 [复制专属邀请链接]

**话术模板（好友收到）**

> 你的朋友 {name} 邀请你使用物价比价 App
> 注册即送 3 天 Pro 体验 👇
> [链接]

**操作流程**

1. 每个注册用户生成唯一的邀请码（`referral_code`），存入 User 表
2. 新用户注册时填写邀请码，邀请人的 `referral_count` +1
3. 当 `referral_count >= 3` 时，自动发放 1 个月 PRO
4. 每月重置阶梯（或设计为累计制）

**Prisma Schema 补充**

```prisma
model User {
  // ... 已有字段
  referralCode   String  @unique @default(cuid())
  referredBy     String?
  referralCount  Int     @default(0)
  proExpiresAt   DateTime?
}
```

### 双方案对比

| 维度 | 方案 A（前 100 名半价） | 方案 B（裂变兑换） |
|------|------------------------|--------------------|
| 启动门槛 | 0 用户即可推 | 需 >= 50 用户 |
| 用户心理 | 稀缺性，FOMO | 社交认同，利他+利益 |
| 技术成本 | 低（一个 Redis 计数器） | 中（邀请码 + 计数逻辑） |
| 预期转化率 | 8-15% | 3-8%（但传播面广） |
| 最佳推出时机 | 上线前 3 天 | 第 2 周 |

**建议执行顺序**：上线时走方案 A 冲注册量 -> 第一周周五开启方案 B 做裂变 -> 两套并行。

- 如果第 3 天注册数 < 10：紧急发一篇 V2EX + 知乎，附上真实截图

- 如果 PRO 转化 = 0：考虑降 PRO 价，或增加免费版核心功能
