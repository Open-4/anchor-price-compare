# 内测用户跟进问卷

> 发送时机：用户注册并完成首次使用后 72 小时，通过邮件或 App 内弹窗触达
> 发送话术开头："用了 3 天物价比价了，想听听你的真实感受——好坏都说，越直白越好。"

---

## 问题 1：如果明天这个 App 停用了，你会有多失望？

- [ ] 非常失望——已经离不开它了
- [ ] 有点失望——但也能接受
- [ ] 不失望——对我没什么影响

**为什么问这个**

这是 Sean Ellis 衡量产品-市场匹配（PMF）的经典问题。"非常失望"的比例超过 40% 说明产品到了可以全力投入的阶段。前 50 个内测用户中有 20 人选"非常失望"就是一个值得庆祝的信号。

---

## 问题 2：在过去 3 天里，你用它查过多少次价格？

- [ ] 0 次——还没用上
- [ ] 1-3 次
- [ ] 4-7 次
- [ ] 8 次以上——基本上每次买东西前都会查

**预期基准**

内测阶段，人均日搜索次数 >= 1 次说明用户养成了使用习惯。如果大部分人选 "0 次" 或 "1-3 次"，需要检查是数据覆盖不够、搜索结果不准，还是用户根本记不住打开。

---

## 问题 3：你搜到过让你"觉得赚到了"的价格信息吗？

- [ ] 有的，在一个具体的商品上发现附近最便宜的店
- [ ] 数据看起来差不多，没觉得有惊喜
- [ ] 搜不到我想买的东西

**为什么问这个**

冷启动阶段最怕用户打开后发现"没有我想要的数据"。"觉得赚到了"是用户留存的核心驱动力——哪怕只发生过一次，用户也会记住这个工具的价值。如果大部分人选了后两项，说明种子数据覆盖的商品品类和用户实际购物清单存在偏差，需要调整数据采集策略。

---

## 问题 4：你愿意为这个 App 每月付多少钱？

- [ ] 不愿意付费
- [ ] 1-5 元/月
- [ ] 6-10 元/月
- [ ] 11-20 元/月
- [ ] 只要不超过 30 元/月，我都愿意

**交叉分析**

将这个问题与问题 1 的回答做交叉：

| 场景 | 解读 | 行动 |
|------|------|------|
| 非常失望 + 愿意付 >= 5 元 | PMF 信号强烈 | 加速 Pro 功能开发，准备正式定价 |
| 非常失望 + 不愿意付费 | 产品有价值但变现阻力大 | 重新设计 Pro 功能权益，突出"值得付"的差异点 |
| 不失望 + 愿意付高价 | 概率极低，可能是用户客气 | 忽略，重点看匹配度高的用户群 |

---

## 问题 5：如果让你向一个朋友介绍这个 App，你会怎么描述它？**请写一句话**

```

[ 文本框 ]

```

**为什么问这个**

这不是一个用来分析的问题——这是用来挖掘卖点的问题。用户的语言最真实。把 50 份回答拿来逐句读，找出高频词和打动人的表述，直接用作首页标题、App Store 描述和朋友圈推广文案。这比任何文案策划都有效。

## 附录：发送方式

### 邮件版（推荐，适合内测用户）

```
主题：用了 3 天物价比价，想听听你的真实感受

正文：

Hi {name}，

我是物价比价的开发者。3 天前你加入了内测，我很好奇你的实际体验。

下面 5 个问题，花 2 分钟答完就好。你的回答会直接影响下一个版本的功能优先级。

[填写问卷 →]

提前感谢！
```

### App 内弹窗版

```
标题：用了几天，感觉怎么样？
描述：2 分钟问卷，你的回答决定下个版本做什么。
按钮：好的，我来说说
关闭：暂时不想
```

---

*这个问卷的核心目标不是收集数据，而是找到那些"非常失望"的前 20 个人——这些人是你最早的传播节点。*
